# 第一波餐厅数据采集计划

## 目标

先用小批量验证采集链路：Google Maps Text Search → 4.0+ 评分过滤 → 去重 → `/api/admin/sync` 拉详情、评论、AI 分析 → 写入本地 D1。

## 第一波范围

- 区域：池袋、上野
- 关键词：湖南料理、湘菜、四川料理、川菜、本格中華、ガチ中華
- 门槛：Google 原始评分 >= 4.0
- 首轮测试数量：6 家以内，按评分和评论数排序

## 执行方式

先预览候选，不写库：

```bash
node scripts/collect-first-wave.mjs --dry-run
```

确认候选后同步入库：

```bash
COLLECT_LIMIT=6 node scripts/collect-first-wave.mjs
```

可扩展区域：

```bash
COLLECT_AREAS="池袋,上野,新宿,高田馬場,秋葉原,神田" COLLECT_LIMIT=20 node scripts/collect-first-wave.mjs
```

可扩展关键词：

```bash
COLLECT_KEYWORDS="湖南料理,湘菜,四川料理,川菜,東北料理,餃子,新疆料理,蘭州牛肉麺,雲南料理,本格中華,ガチ中華" node scripts/collect-first-wave.mjs
```

## 后续采集节奏

1. 每次新增 2-4 个区域，不超过 20 家，观察 Google Maps 和 MiniMax 成本。
2. 每轮先 dry-run，看候选是否明显跑偏，再正式同步。
3. 同步后检查 `/zh/restaurants` 和 `/zh/map`，确认坐标、评分、摘要、菜系是否正常。
4. 对 AI 菜系误判的餐厅，后续增加人工修正字段或后台审核入口。
5. 稳定后把脚本接入 Cloudflare Cron，每周只同步已入库餐厅的新评论。
