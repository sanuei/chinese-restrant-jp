# 🍜 真味中华 / ガチ中華ナビ —— 在日中国餐厅权威评鉴平台

**中文副标题**：在日中国餐厅权威评鉴  
**日语名称**：ガチ中華ナビ（"ガチ" = 真正、正宗）  
**产品定位**：双语（中文/日文）平台，面向在日华人 & 对正宗中华料理感兴趣的日本人，提供经 AI 过滤虚假评论、按菜系分类、标注正宗度的中国餐厅评鉴导航。

---

## 一、产品概述

### 核心价值主张

> 中文：「在异乡，让每一口都像回家」  
> 日文：「本物の中華を、あなたの街で。」

- **真实评分**：AI 智能过滤虚假评论，呈现可信加权评分
- **菜系分类**：按八大菜系精准分类，找到家乡味道
- **正宗认证**：三级标签区分"正宗中华" / "改良中华" / "日式中华"
- **AI 摘要**：双语自动生成每家餐厅的评论精华摘要
- **起步区域**：先聚焦东京都 23 区，精耕后再扩展

### 模块 3：AI 虚假评论检测（MiniMax）⭐ 核心竞争力

#### 虚假信号识别
```
降权或排除：
  - 文本过短且无实质内容（"美味しい！★5"）
  - 账号首条评论或极少活跃记录
  - 同一时间段大量五星/一星评论涌入（刷单/恶意差评）
  - 评论措辞与账号其他评论风格明显不符
  - 与餐厅同区域竞争者账号交叉出现

提权信号：
  - 提及具体菜名、价格、位置细节
  - 有合理批评与夸奖混合
  - 评论者有跨多家餐厅的活跃历史
  - 附有真实照片
  - 时间分布自然（非集中爆发）
```

#### 可信评分公式
```
可信评分 = Σ(评论原始分 × 可信权重) / Σ(可信权重)

权重分级：
  可信评论：1.0
  存疑评论：0.3
  确认虚假：0（排除）

最终展示：可信加权评分 ± 置信区间
```

#### MiniMax 调用设计
```
Prompt 策略：
  - 输入：评论文本（中/日/英三语支持）+ 账号元数据
  - 输出：{ credibility: 0-100, reason: "...", action: "keep/flag/remove" }
  - 模型：MiniMax abab6.5s（性价比高，适合批量处理）
  - 批量处理：每次最多50条评论，节省 API 费用
```

### 模块 5：正宗认证系统 ⭐ 差异化核心

#### 判定标准

| 判定维度 | 正宗中华 ✅ | 改良中华 🟡 | 日式中华 🔵 |
|---------|-----------|-----------|-----------|
| 经营者背景 | 华人/中国人 | 华人但日式经营 | 日本人 |
| 菜单语言 | 含中文 | 中日双语 | 纯日文 |
| 调味风格 | 原产地风味 | 适度日本化 | 大幅减辣减咸 |
| 评论关键词 | "地道""家乡味""正宗" | 混合 | "あっさり""食べやすい" |
| 厨师来源 | 中国大陆/港澳台 | 混合 | 日本培训 |

#### 三级标签
```
🔴 ガチ中華（正宗中华）  — 4-5项满足，来自中国的地道口味
🟡 アレンジ中華（改良中华）— 2-3项满足，融合本地化改良
🔵 日式中華（日式中华）  — 0-1项满足，拉面/饺子等已日式化品类
```

> **重要说明**：三级标签是信息工具，不是价值判断。  
> 「正宗」帮助华人找家乡味，「日式中华」帮助日本人找熟悉口味，各有受众。

#### 认证数据来源
1. **AI 自动判断**：基于 Google Maps 评论关键词 + 餐厅名称
2. **用户众包**：注册用户可提交补充信息（图片菜单、厨师背景）
3. **申诉机制**：餐厅方可通过表单申诉，平台人工复核（阶段2上线）

## 五、数据库设计（D1 SQLite）

### 核心表结构

```sql
-- 餐厅主表
CREATE TABLE restaurants (
  id TEXT PRIMARY KEY,              -- Google place_id
  name_zh TEXT,                     -- 中文名
  name_ja TEXT,                     -- 日文名
  address TEXT,
  lat REAL, lng REAL,
  cuisine_type TEXT,                -- 菜系标签
  authenticity TEXT,                -- 'authentic' | 'adapted' | 'japanese'
  authenticity_score INTEGER,       -- 0-100
  raw_rating REAL,                  -- Google原始评分
  trusted_rating REAL,              -- AI可信加权评分
  review_count INTEGER,
  trusted_review_count INTEGER,
  price_level INTEGER,              -- 1-4
  ai_summary_zh TEXT,               -- 中文AI摘要
  ai_summary_ja TEXT,               -- 日文AI摘要
  last_synced_at TEXT,
  created_at TEXT
);

-- 评论表
CREATE TABLE reviews (
  id TEXT PRIMARY KEY,
  restaurant_id TEXT,
  author_name TEXT,
  rating INTEGER,
  text TEXT,
  language TEXT,                    -- 'zh' | 'ja' | 'en'
  credibility_score INTEGER,        -- 0-100
  credibility_action TEXT,          -- 'keep' | 'flag' | 'remove'
  credibility_reason TEXT,
  published_at TEXT,
  FOREIGN KEY(restaurant_id) REFERENCES restaurants(id)
);

-- 全文搜索虚拟表
CREATE VIRTUAL TABLE restaurants_fts USING fts5(
  name_zh, name_ja, address, cuisine_type
);
```

## 六、页面规划

### URL 结构（双语 i18n）

```
/                    → 首页（Hero + 菜系入口 + 热门推荐）
/[locale]/           → /zh/ 或 /ja/ 语言切换

/about               → 关于 + 认证标准说明
```

### 关键页面设计

#### 首页
- Hero：大图背景 + 双语标题 + 搜索框（支持餐厅名/菜系/区域）
- 菜系快速入口：9大菜系图标横排
- 认证说明：ガチ🔴 / アレンジ🟡 / 日式🔵 三色 Banner 解释
- 热门餐厅：东京 Top10 横向滚动卡片

#### 餐厅列表页
- 左侧筛选：区域（23区）、菜系、认证类型、评分≥X、价格
- 右侧：卡片列表 + 右上角小地图同步高亮
- 排序：可信评分 / 评论数 / 距离 / 最新
- 卡片显示：认证徽章、菜系标签、AI摘要一句话、两种评分对比（原始 vs 可信）

#### 餐厅详情页
- 顶部：照片轮播（来自 R2 镜像的 Google 照片）
- 认证状态 + 判定理由（可展开）
- AI 双语摘要卡片（可切换语言）
- 评分对比图：原始评分 vs 可信评分（差距大时显示警示）
- 真实评论列表（按可信度排序）
- 底部：过滤透明度（显示"已过滤 X 条低可信评论"）
